## Body

### 들어가며

AFH(Adult Family Home) 운영을 준비할 때 **Initial Inspection**이 가장 큰 관문 중 하나입니다.  
체크리스트, WAC·RCW, 정책서·케어플랜·MAR, 점검관 Q&A까지 준비할 것이 많고, **실무형 가이드**가 없으면 시간과 부담이 크게 늘어납니다.

**AFH Moltbot Consultant Curriculum**은 이 갭을 메우기 위한 **단일 통합 교육 포털**입니다.  
대상은 AFH **컨설턴트·어드바이저**이고, **Moltbot(Clawdbot)**을 이용해 점검 준비 서류·논리의 **80~90%를 AI에게 위임**하는 방법을 단계별로 배우게 됩니다.

---

### 포털 구성

| 메뉴 | 내용 |
|------|------|
| **Curriculum** | 4단계 커리큘럼 + 모듈(환경 설정, WAC 주입, 서류 자동화, 논리·논변, 플레이북) |
| **Moltbot Scripts** | 상위 10개 CLI 명령 + AFH 자동화 예시 |
| **Inspection Prompt Pack** | 50개 이상 실전 프롬프트 (물리적 환경·서류·점검관 대응) |
| **Paperwork Binder** | 01_Compliance_Brain ~ 04_Operations 표준 폴더 구조 |
| **AI Playbooks** | 제로터치 스크립트, NCP 초안, 점검 당일 실시간 방어, 3대 원칙 |
| **AI Use Boundaries** | 허용/금지, 윤리·규정 가이드 |
| **Top 100 Questions** | 5개 카테고리 100개 질문 + AI 사고 흐름 샘플 |

---

### 왜 Moltbot인가?

- **WAC·RCW·DSHS**를 “지식”으로 넣어두면, 점검관이 물을 때 **WAC 번호 + 논리적 답변**을 즉시 생성할 수 있습니다.
- **정책서·케어플랜·MAR 감사·입소 계약서** 등 반복 서류는 **프롬프트 한 번**으로 초안을 만들 수 있고, 사람은 **검토·편집**만 하면 됩니다.
- 점검 당일에는 **텔레그램/WhatsApp**으로 Moltbot에 질의해, 현장에서 “WAC 몇 조 몇 항에 따라…” 하고 **당당하게** 답할 수 있습니다.

### 시작하기

- **퀵스타터**: 5분 안에 포털 실행 (`npm install` → `npm run dev`).
- **튜토리얼**: 단계별로 폴더 구조 만들기, Moltbot 설정, 프롬프트·질문 bank 연습.
- **매뉴얼**: 전체 기능·구성·사용 방법 정리.

프로젝트 저장소/배포 URL에서 **README·docs 폴더**를 보시면 됩니다.

---

### 정리

- AFH Initial Inspection 준비를 **체계적·자동화**하고 싶다면 → 커리큘럼·프롬프트·플레이북을 한 번에 쓸 수 있는 포털.
- Moltbot을 **AFH 규정·서류**에 활용하고 싶다면 → 스크립트·프롬프트·바인더 구조가 정리되어 있음.
- 점검관 Q&A를 **WAC 인용**으로 대비하고 싶다면 → Top 100 질문 + AI 사고 흐름으로 연습 가능.
